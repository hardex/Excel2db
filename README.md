# Excel2db

Local web app for creating and editing Excel-mapping rules (Stage 1 of a two-stage Excel-to-CSV pipeline). No database, no auth — file-based storage only. The companion Stage 2 batch processor lives in [`excel_processor`](https://github.com/hardex/excel_processor).

## Quick start

```bash
git clone https://github.com/hardex/Excel2db.git
cd Excel2db
python3 -m venv .venv
source .venv/bin/activate           # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
# → http://127.0.0.1:8000
```

Requires Python 3.12+.

## Tech stack

- **Backend**: Python 3.12, FastAPI, Pydantic
- **Excel**: openpyxl
- **Frontend**: Jinja2 server-rendered HTML, vanilla JS, vanilla CSS

## Project Structure

```
app/
  main.py                  # FastAPI entry, mounts routers + static
  models/schemas.py        # FieldModel, TemplateModel (Pydantic)
  routes/
    templates.py           # Template CRUD + test-cell + check-cell (AJAX)
    processing.py          # Upload+process (single step), correction, result
    logs.py                # Log viewer
    validation.py          # Reserved (empty)
  services/
    template_service.py    # Read/write JSON files in mappings/
    excel_service.py       # openpyxl reader (dual-mode: raw + calculated)
    validation_service.py  # Field validation (string/number/date + allow_empty)
    output_service.py      # Generate output JSON to outputs/
    logging_service.py     # Python logging to logs/app.log
  templates/               # Jinja2 HTML
  static/css/style.css
  static/js/app.js
mappings/                  # Template JSON: {code}_{version}.json
uploads/                   # Temp uploaded .xlsx files
outputs/                   # Final JSON output
logs/                      # app.log
```

## Core Models

**FieldModel**: `field_code`, `field_name`, `sheet`, `cell`, `value_type` (string|number|date), `allow_empty`, `active`, `raw_cell_value`, `description`

**TemplateModel**: `template_code`, `template_version`, `description`, `is_default`, `fields[]`

## Processing Flow

Single-step: user picks file + template → POST /process/start → extract → validate → if errors: correction page → revalidate → if all valid: generate JSON → result page with download.

Session is in-memory dict keyed by UUID cookie. Discarded on page exit.

## Excel Reading

`read_workbook_fields()` opens workbook in two modes only when needed:
- `data_only=False` for fields with `raw_cell_value=True` (formulas as text)
- `data_only=True` for fields with `raw_cell_value=False` (calculated values)

## Validation Rules

- **Empty**: invalid only when `None` or `""`. Spaces-only `"   "` is valid.
- **Number**: regex `^-?(\d{1,3}(,\d{3})*|\d+)(\.\d+)?$`
- **Date**: native datetime OR text: YYYY-MM-DD, DD.MM.YYYY, MM/DD/YYYY, DD/MM/YYYY
- **String**: always valid, only allow_empty applies
- No auto-normalization. No trimming. Values preserved as-is.

## Template Editor (template_edit.html)

Heavy JS page with client-side state:
- `fields[]` array + `editingIndex` (null=add, number=edit)
- Fields table above add/edit form. Double-click row to edit.
- Hidden inputs generated by JS for form submission (field_code_N, field_name_N, etc.)
- Inline "Check" button: AJAX POST to `/templates/check-cell` — reads cell + validates type
- Sheet and test file inputs persist between field additions

## Form Patterns

- Template fields parsed by `_parse_fields_from_form()`: iterates index 0..N until `field_code_{index}` is None
- Checkboxes: presence with value in ("on","true","1") = True
- Flash messages via query params: `?msg=...&msg_type=success|error`
- Redirects use 303 status code

## Key Business Rules

1. Only `.xlsx` files
2. No automatic data modification
3. Final JSON blocked until all validation passes
4. Only invalid fields are editable on correction page
5. All corrections logged with original + corrected values
6. One template can be marked as default (preselected)

## CSS Conventions

- `.form-section` white card, `.form-grid` CSS grid, `.form-group` flex column
- `.btn--primary|secondary|danger`, `.btn--sm`
- `.flash--success|error|info`
- `.fields-table` for template editor, `.data-table` for lists
- `.check-ok|check-err` for cell check result styling
- `.row-editing` + `.editing-mode` for edit state highlighting
